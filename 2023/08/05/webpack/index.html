概念

  • webpack是前端项目工程化的具体解决方案
  • 提供友好前端模块化开发支持，代码压缩混淆，处理浏览器端js兼容性，性能优化
  • 提高开发效率和项目可维护性

npm init -y

可在当前目录创建一个 package.json 文件

安装

npm install webpack@5.42.1 webpack-cli@4.7.2 -D

-D 开发阶段 -S 开发和上线阶段都用到，package.json里有相应节点

创建webpack.config.js

真正运行之前，先读取webpack.config.js,然后拿到配置选项

在 webpack 4.x 和 5.x 版本中，默认约定：

  • 默认打包入口文件为 src/index.j`
  • 默认输出文件路径为 dist/main.js

webpack-dev-server

监听项目源码的变化，从而自动打包构建,会启动一个实时的http服务器

安装

html-webpack-plugin

将webpack打包后输出的所有脚本文件自动添加到插件生成的HTML文件中

安装

webpack.config.js

loader

webpack 默认只能打包 .js 结尾的模块，其他非.js模块需要调用对应的 loader 加载器才能进行打包

css-loade 可打包 css文件

安装

npm i style-loader@3.0.0 css-loader@5.2.6 -D

webpack.config.js

less-loader`可打包 .less文件

安装·

npm i less-loader@10.0.1 less@4.1.1 -D

webpack.config.js

label-loade 可打包 webpack 无法处理的 JS 高级语法

npm i babel-loader@8.2.2 @babel/core@7.14.6 @babel/plugin-proposal-decorators@7.14.5 -D

babel.config.js
