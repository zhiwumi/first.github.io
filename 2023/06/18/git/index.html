Git介绍

开源的分布式版本控制系统，版本控制系统在项目开发中作用重大，主要的功能有以下几点

  • 代码备份
  • 版本回退
  • 协作开发
  • 权限控制

Git快照是在原有文件版本的基础上重新生成一份新的文件，类似备份，在版本切换时非常快。

三个区域

工作区，暂存区，Git仓库(最终存放的区域)

三种状态

已修改，已暂存，已提交

配置账户

全局配置，对所有Git仓库有效

局部，只对当前Git仓库有效

初始化为Git仓库,会创建一个.git的隐藏目录，此目录为当前项目的Git仓库

将本地仓库与远程仓库进行关联

地址最好为SSH，不需要验证

将本地代码推送到仓库中，第一次提交需要加 -u参数后，后面的提交就直接可以 git push

项目变化后，在本地进行提交之后，再同步到远程仓库中

①：执行 add 命令，将代码添加到暂存区
②：执行 commit 命令，将代码提交到本地仓库。（因为只有本地仓库的代码才能推送到远程）
③：执行 git push 命令，将这次改动推送到远程仓库。

推送本地指定分支到仓库指定分支

第一次 -u，冒号后面为对本地分支的重命名，不需要重命名，可把冒号及后面删掉,别名（origin）
